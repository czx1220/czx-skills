# 每日进展总结

通用日报 skill：扫描当天产出 → 展示提取到的数据 → 与用户确认结论 → 生成日报。

## 配置（按当前项目调整）

首次在某项目使用时，先和用户确认以下信息（同会话后续沿用）：

- **产出目录**：当天的模型/实验/结果存放在哪（如 `$MODELS_DIR`、`$RESULTS_DIR`）
- **筛选规则**：如何识别"今天的"产出（按日期目录、文件名前缀、mtime 等）
- **关键指标**：本项目关注哪些 metric（如 mIoU / Acc / loss），以及对比基线（baseline）的数值
- **日报输出路径**：默认写到 `/tmp/daily_summary_YYYYMMDD.md`；用户要保留则写到项目目录下自定义目录

若用户未指定，用合理默认并在日报里注明假设。

## 执行流程（两阶段）

### 阶段一：提取数据 + 与用户交流

1. 扫描今日产出（示例，按配置替换路径与规则）：
```bash
TODAY=$(date +%Y%m%d)
# 例：按日期 + 项目前缀筛选今日模型/实验目录
ls "$MODELS_DIR" | grep "$RUN_PREFIX" | grep "$TODAY"
```

2. 读取每个产出的参数与评测结果（示例）：
```bash
cat "$MODELS_DIR/<产出目录>/hparams.txt"
cat "$RESULTS_DIR/<对应结果目录>/summary.txt"
```

3. 如有横向对比表，一并读取：
```bash
cat "$RESULTS_DIR/comparison.txt"
```

4. **向用户展示提取到的数据（表格），然后询问**：
   - 今天核心探索的问题是什么？
   - 得出了什么结论？
   - 下一步打算做什么？

### 阶段二：根据用户反馈生成日报

将用户确认的结论和下一步计划，结合提取的数据，写入日报文件。

## 输出格式

```markdown
### YYYY-MM-DD [核心主题]

#### 今日进展

[用户确认的核心问题和结论，配上数据表格]

| 方案 | 参数 | [指标1] | [指标2] | vs baseline |
|------|------|---------|---------|-------------|

[结论]

#### 下一步

- [ ] 具体实验1（带参数）
- [ ] 具体实验2（带参数）
```

## 输出要求

- 默认写入 `/tmp/daily_summary_YYYYMMDD.md`；用户要长期保留则写到项目目录下自定义目录（如 `outputs/`）
- `###` 用于日期+主题标题行，`####` 用于「今日进展」和「下一步」两个子节
- 对比列以项目配置的 baseline 数值为准
- 「下一步」用 `- [ ]` todo 格式，具体可跑的实验带参数
- **结论和下一步必须经过用户确认，不要自行推断**
- 表格里只填实际从文件提取到的数据，缺失项留空或标注"未测"，不要编造数值
